## Purpose

HTTP API for MCP (Model Context Protocol) server lifecycle management. MCP servers provide tools to the agent. The service maintains a registry of connected MCP servers.

## ADDED Requirements

### Requirement: Connect MCP server

The service SHALL provide `POST /api/v1/mcp/connect` accepting `{"serverId": "...", "config": {"command": "npx", "args": ["-y", "@modelcontextprotocol/server-filesystem", "/workspace"], "env": {...}}}`. The server SHALL be connected via `connectMCPServer(config)` and tracked by `serverId`.

#### Scenario: Connect MCP server
- **WHEN** caller sends `POST /api/v1/mcp/connect` with valid MCP server config
- **THEN** the MCP server process is spawned
- **THEN** the server is registered by serverId
- **THEN** response is `{"ok": true, "serverId": "..."}`

#### Scenario: Connect MCP server without serverId
- **WHEN** caller sends `POST /api/v1/mcp/connect` without `serverId`
- **THEN** a serverId is auto-generated (UUID)
- **THEN** response is `{"ok": true, "serverId": "<auto-generated>"}`

#### Scenario: Connect duplicate serverId
- **WHEN** caller sends `POST /api/v1/mcp/connect` with a `serverId` that already exists
- **THEN** the existing server is replaced with the new connection
- **THEN** old server is disconnected first

### Requirement: List MCP tools

The service SHALL provide `GET /api/v1/mcp/tools` returning all available tools from all connected MCP servers, formatted as tool definitions.

#### Scenario: List tools from connected servers
- **WHEN** caller sends `GET /api/v1/mcp/tools`
- **THEN** response is `{"tools": [{"name": "mcp__server__toolname", "description": "...", "inputSchema": {...}}, ...]}`

#### Scenario: List tools when no servers connected
- **WHEN** caller sends `GET /api/v1/mcp/tools` with no MCP servers connected
- **THEN** response is `{"tools": []}`

### Requirement: Call MCP tool directly

The service SHALL provide `POST /api/v1/mcp/tools/call` accepting `{"serverId": "...", "toolName": "...", "toolInput": {...}}` for calling a specific MCP tool outside of agent execution.

#### Scenario: Call MCP tool directly
- **WHEN** caller sends `POST /api/v1/mcp/tools/call` with valid serverId, toolName, and input
- **THEN** the tool is called via the MCP protocol
- **THEN** response is `{"result": "...", "isError": false}`

#### Scenario: Call tool on nonexistent server
- **WHEN** caller sends `POST /api/v1/mcp/tools/call` with unknown serverId
- **THEN** response is `404 Not Found` with `{"error": "MCP server not found"}`

### Requirement: Disconnect MCP server

The service SHALL provide `DELETE /api/v1/mcp/:serverId` to disconnect and clean up an MCP server.

#### Scenario: Disconnect existing server
- **WHEN** caller sends `DELETE /api/v1/mcp/my-server`
- **THEN** the MCP server process is terminated
- **THEN** the server is removed from the registry
- **THEN** response is `{"ok": true}`

#### Scenario: Disconnect nonexistent server
- **WHEN** caller sends `DELETE /api/v1/mcp/nonexistent`
- **THEN** response is `404 Not Found` with `{"error": "MCP server not found"}`

### Requirement: Service disconnect all MCP servers on shutdown

During graceful shutdown, the service SHALL disconnect all connected MCP servers before exiting.

#### Scenario: Shutdown disconnects all MCP servers
- **WHEN** service receives SIGTERM
- **THEN** all connected MCP servers are disconnected
- **THEN** no zombie MCP server processes remain
